Overview

Select Sense is a powerful Chrome extension designed to streamline web element identification and test automation. Whether you're a seasoned automation engineer or just getting started with web testing, our tool provides everything you need to efficiently locate elements and generate test code.

Key Benefits:
  • Instantly generate accurate locators for any web element
  • Create Page Object Models with a single click
  • Test locators in real-time
  • Save and manage your locators
  • Generate code snippets in multiple languages

Features

Smart Element Selection

Click on any element to instantly generate optimal locators using multiple strategies.

Multiple Locator Strategies

Get CSS, XPath, ID, name, class, and link text locators automatically.

Real-time Testing

Test your locators instantly with visual feedback.

Code Generation

Generate code snippets in Java, Python, JavaScript, and more.

Page Objects

Create complete Page Object Models for your test automation framework.

Library Management

Save and organize your locators for future use.

Getting Started

1. Installation

Install the Select Sense extension from the Chrome Web Store.

2. Activation

Click the extension icon in your browser to activate it on any webpage.

3. Element Selection

Click on any element on the page to generate locators.

4. Using the Panel

Use the locator panel to view, test, and save your locators.

How to Use Each Feature

Element Selection

To select an element:

  1. Activate the extension
  2. Move your mouse over the desired element
  3. Click to generate locators

Testing Locators

To test a locator:

  1. Select a locator strategy (CSS, XPath, etc.)
  2. Click the "Test" button
  3. The element will be highlighted if found

Saving Locators

To save locators:

  1. Click the "Save" button
  2. Choose between Library or File download
  3. Enter a name for your locator
  4. Select the format (JSON/CSV)

Generating Code

To generate code snippets:

  1. Click the "Code Snippets" button
  2. Select your preferred programming language
  3. Copy the generated code

Creating Page Objects

To generate a Page Object Model:

  1. Save multiple elements to a collection
  2. Click the Page Object icon
  3. Select your framework and language
  4. Customize the class name and settings

XPath Tutorial

Basic XPath Syntax

Syntax Description Example
//tagname Find any element with the tag //button
[@attribute='value'] Find by attribute //input[@type='submit']
[text()='text'] Find by exact text //button[text()='Submit']
[contains()] Find by partial match //div[contains(@class, 'menu')]

Creating Custom XPath

1. Start with the Element Type

//button

2. Add Attributes

//button[@class='submit-btn'][@type='submit']

3. Use Contains for Dynamic Values

//button[contains(@class, 'btn')][contains(text(), 'Submit')]
Tip: Use the extension's Test feature to verify your XPath works correctly.

CSS Selector Tutorial

Basic CSS Selector Syntax

Syntax Description Example
#id Select by ID #submit-button
.class Select by class .btn-primary
[attribute=value] Select by attribute [type="submit"]
parent > child Direct child .form > button

Creating Custom CSS Selectors

1. Start with Element Type or Class

button.submit-btn

2. Add Attributes

button.submit-btn[type="submit"]

3. Use Parent-Child Relationships

.form-container > .input-group > button.submit-btn
Tip: CSS selectors are often more readable than XPath and should be your first choice when possible.

Testing Your Locators

Using the Test Feature

1. Enter Your Locator

Type or paste your custom locator in the appropriate field (CSS or XPath).

2. Select Strategy

Choose the correct strategy (CSS, XPath, etc.) for your locator.

3. Click Test

The extension will highlight the matching element if found.

Common Issues:
  • Check for typos in your selector
  • Verify quotes are properly matched
  • Ensure elements aren't in iframes
  • Check if elements are dynamically loaded

Best Practices

  • Start with simple selectors and add specificity as needed
  • Use meaningful IDs and classes when available
  • Test locators across different states of the application
  • Consider dynamic content and page changes
  • Save working locators to the library for future use
